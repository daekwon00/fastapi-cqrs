# FastAPI CQRS 구현 계획서

> **작성일**: 2026-03-09
> **프로젝트**: Spring Boot → FastAPI 마이그레이션 (CQRS 패턴)
> **환경**: Cloud MSA 구조 (로컬 개발 우선)
> **원본**: `spring-cqrs` (Spring Boot 4 + Java 25 + MyBatis + PostgreSQL)
> **대상**: `fastapi-cqrs` (FastAPI + Python 3.12+ + asyncpg + Pydantic v2)

---

## 1. 프로젝트 개요

### 1.1 목적

기존 Spring Boot 기반 인트라넷 API를 FastAPI로 재구현하여, Python 생태계 기반의 경량 MSA 서비스를 구축한다. 동일한 PostgreSQL DB(intranet/internal 스키마)를 공유하며, 기존 테이블 구조와 70개 SQL 쿼리를 그대로 유지한다.

### 1.2 기술 스택


| 구분     | Spring Boot (AS-IS)    | FastAPI (TO-BE)               |
| ------ | ---------------------- | ----------------------------- |
| 프레임워크  | Spring Boot 4.0.3      | FastAPI + Uvicorn             |
| 언어     | Java 25                | Python 3.12+                  |
| DB 접근  | MyBatis (동기)           | asyncpg (비동기, ORM 미사용)        |
| 스키마 검증 | Jakarta Validation     | Pydantic v2                   |
| 인증     | Spring Security + JJWT | python-jose + passlib[bcrypt] |
| AI 통신  | Java HttpClient        | httpx (비동기)                   |
| SSE    | SseEmitter             | sse-starlette                 |
| 설정 관리  | application.yaml       | pydantic-settings + .env      |


### 1.3 아키텍처 패턴 매핑

```
[Spring Boot]                        [FastAPI]
Controller (@RestController)    →    Router (APIRouter)
Service (@Service)              →    Service class (Depends 주입)
MyBatis Mapper XML              →    Repository class (asyncpg raw SQL)
Java Record DTO                 →    Pydantic v2 BaseModel
Spring Security Filter          →    FastAPI middleware + Depends
@Value / application.yaml       →    pydantic-settings + .env
HikariCP 커넥션 풀              →    asyncpg.create_pool()
```

### 1.4 CQRS 패턴 유지

```
[Command 경로] Router → CommandService → CommandRepository (INSERT/UPDATE/DELETE)
[Query 경로]   Router → QueryService   → QueryRepository   (SELECT)
```

## 4. SQL 쿼리 이관 현황

총 **70개 쿼리** (Command 34, Query 36)


| 모듈                 | Command | Query | 합계  | 난이도                 |
| ------------------ | ------- | ----- | --- | ------------------- |
| auth               | 4       | 3     | 7   | 중                   |
| user               | 2       | 1     | 3   | 하                   |
| board (board+post) | 6       | 6     | 12  | **상** (동적 검색)       |
| file               | 1       | 1     | 2   | 하                   |
| dashboard          | 0       | 5     | 5   | 중 (generate_series) |
| admin/user         | 5       | 3     | 8   | 중                   |
| admin/board        | 3       | 2     | 5   | 하                   |
| admin/dashboard    | 0       | 5     | 5   | 중                   |
| system (5개 하위)     | 13      | 10    | 23  | 중                   |


### MyBatis → asyncpg 변환 규칙


| MyBatis             | asyncpg                         |
| ------------------- | ------------------------------- |
| `#{param}`          | `$1, $2, ...` (positional)      |
| `${param}` (문자열 치환) | Python f-string + **화이트리스트 검증** |
| `<if test="...">`   | Python `if` + 쿼리 문자열 연결         |
| `<foreach>` IN절     | `ANY($1::text[])`               |
| `RETURNING post_id` | `await conn.fetchval()`         |

### Phase 1: 보안 + Auth 모듈


| 작업           | 설명                                            |
| ------------ | --------------------------------------------- |
| JWT Provider | HS256, access(1시간)/refresh(7일) 토큰             |
| 패스워드 유틸      | BCrypt 해싱 + 평문 패스워드 fallback                  |

## 8. Cloud MSA 확장 고려사항

현재 **모놀리식**으로 개발하되, 모듈별 독립 패키지 구조로 향후 분리 가능:


| 현재 (로컬)            | 향후 (Cloud MSA)         |
| ------------------ | ---------------------- |
| 단일 FastAPI 앱       | 모듈별 독립 마이크로서비스         |
| common/security 내장 | 공유 라이브러리 (pip 패키지)     |
| CORS 설정            | API Gateway 레벨 라우팅     |
| 동기 호출              | 이벤트 드리븐 (서비스 간 이벤트 발행) |
| docker-compose     | Kubernetes 오케스트레이션     |

## 9. 실행 계획 (세션 단위)


| 세션     | 범위        | 핵심 산출물                             |
| ------ | --------- | ---------------------------------- |
| **1차** | Phase 0~3 | 스캐폴딩 + 인증 + 사용자 + 게시판 (핵심 CRUD 완성) |
| **2차** | Phase 4~6 | 파일 + 대시보드 + AI 채팅                  |
| **3차** | Phase 7~9 | Admin + System + 통합 마무리            |


### 진행 방식

1. Phase 단위로 구현 → 검증 → 커밋
2. 각 Phase 완료 시 `uvicorn` 실행 검증
3. 동일 DB로 Spring Boot API와 병행 테스트 가능
4. 커밋 메시지는 한글로 작성, 사용자 승인 후 커밋
