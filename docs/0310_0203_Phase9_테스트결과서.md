## 테스트 아키텍처

### fixture 구성 (conftest.py)

| fixture | scope | 설명 |
|---|---|---|
| `client` | session | httpx.AsyncClient + ASGITransport, DB pool 수동 관리 |
| `test_user_id` | session | UUID 기반 고유 테스트 사용자 ID |
| `registered_user` | session | register API로 사용자 등록, teardown 시 DB 삭제 |
| `user_token` | session | login API로 발급받은 access_token |
| `user_headers` | session | `Authorization: Bearer {token}` 헤더 |
| `admin_token` | session | `create_access_token("admin", ["ROLE_ADMIN"])` 직접 생성 |
| `admin_headers` | session | 관리자 인증 헤더 |
| `override_upload_dir` | session (autouse) | FILE_UPLOAD_DIR을 임시 경로로 변경 |

### 테스트 방식
- **httpx.ASGITransport**: uvicorn 서버 기동 없이 ASGI 앱 직접 호출
- **session-scoped event loop**: 모든 테스트가 단일 이벤트 루프에서 실행
- **실제 DB 사용**: mock 없이 PostgreSQL에 직접 쿼리 (통합 테스트)
- **테스트 데이터 정리**: 테스트 사용자는 teardown 시 DB에서 삭제

## 구현 중 발견/수정한 이슈

| 이슈 | 원인 | 해결 |
|---|---|---|
| passlib + bcrypt 5.x 비호환 | passlib이 bcrypt 5.x의 API 변경을 미지원 | `app/security/password.py`에서 bcrypt 직접 사용으로 전환 |
| 비존재 사용자 로그인 시 500 | `tb_login_history.user_id` FK 제약 위반 | `_record_login_history`에 try/except 추가 |
| event loop 불일치 | session fixture와 function-scoped 테스트 루프 분리 | `asyncio_default_test_loop_scope = "session"` 설정 |
