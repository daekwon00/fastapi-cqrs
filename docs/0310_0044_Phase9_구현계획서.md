# Phase 9: 통합 테스트 + 마무리 구현 계획

## Context
FastAPI CQRS 프로젝트의 Phase 0~8이 완료되어 9개 도메인 모듈, 48개 API 엔드포인트, 70개 SQL 쿼리가 구현된 상태. Phase 9에서는 통합 테스트 작성, 요청 로깅 미들웨어 추가, README 최종 업데이트를 수행한다. PostgreSQL(localhost:5432) 실행이 필요하며, FastAPI 서버(uvicorn) 기동은 불필요하다.

---

## 구현 순서

### Step 1: pyproject.toml 수정
- `testpaths = ["tests"]` 추가
- pytest-asyncio 설정 보완 (loop_scope)

### Step 2: tests/conftest.py (핵심 fixture)
**DB pool 수동 관리** (httpx.ASGITransport는 lifespan 미지원):
```python
@pytest.fixture(scope="session")
async def client():
    from app.database import create_pool, close_pool
    await create_pool()
    transport = httpx.ASGITransport(app=app)
    async with httpx.AsyncClient(transport=transport, base_url="http://testserver") as c:
        yield c
    await close_pool()
```

**fixture 목록**:
| fixture | scope | 설명 |
|---------|-------|------|
| `client` | session | httpx.AsyncClient + ASGITransport, DB pool 생성/해제 |
| `test_user_id` | session | UUID 기반 테스트 사용자 ID |
| `registered_user` | session | register 엔드포인트로 사용자 등록 + teardown 시 DB 삭제 |
| `user_token` | session | login으로 발급받은 access_token |
| `user_headers` | session | `{"Authorization": "Bearer {token}"}` |
| `admin_token` | session | `create_access_token("admin", ["ROLE_ADMIN"])` 직접 생성 |
| `admin_headers` | session | 관리자 인증 헤더 |
| `override_upload_dir` | session, autouse | FILE_UPLOAD_DIR을 임시 경로로 변경 |

### Step 3~12: 테스트 파일 작성 (우선순위순)

#### 3. tests/test_health.py (1개 테스트)
- `test_health_check`: GET /health → 200

#### 4. tests/test_auth.py (7개 테스트)
- `test_register_success`: POST /api/v1/auth/register → 201
- `test_register_duplicate`: 중복 username → 에러
- `test_login_success`: POST /api/v1/auth/login → 200, accessToken/refreshToken
- `test_login_wrong_password`: 잘못된 비밀번호 → 401
- `test_login_nonexistent_user`: 없는 사용자 → 401
- `test_refresh_token`: POST /api/v1/auth/refresh → 200, 새 accessToken
- `test_refresh_invalid_token`: 잘못된 토큰 → 401

**주의**: RefreshRequest는 `alias="refreshToken"` 사용 → JSON에 `refreshToken` 키 필요

#### 5. tests/test_board.py (9개 테스트)
- 클래스로 묶어 CRUD 순서 보장: create → read → update → delete
- `test_get_board_list`: GET /api/v1/boards → 200
- `test_get_board_by_id`: GET /api/v1/boards/{id} → 200
- `test_get_board_not_found`: 없는 ID → 404
- `test_create_post`: POST /api/v1/posts → 201 (인증 필요)
- `test_get_post_detail`: GET /api/v1/posts/{id} → 200
- `test_get_post_list_pagination`: GET /api/v1/boards/{id}/posts?page=1&size=5 → PageResponse
- `test_update_post`: PUT /api/v1/posts/{id} → 200
- `test_delete_post`: DELETE /api/v1/posts/{id} → 200
- `test_get_recent_posts`: GET /api/v1/posts/recent → 200

**참고**: CreatePostRequest에 `board_id`(alias: boardId), `file_ids`(alias: fileIds) 존재

#### 6. tests/test_user.py (4개 테스트)
- `test_get_my_profile`: GET /api/v1/users/me → 200
- `test_update_my_profile`: PUT /api/v1/users/me → 200
- `test_change_password`: PUT /api/v1/users/me/password → 200
- `test_get_profile_unauthorized`: 토큰 없이 → 401

#### 7. tests/test_file.py (3개 테스트)

#### 8. tests/test_admin.py (6개 테스트)

- `test_admin_get_user_by_id`: GET /api/v1/admin/users/{id} → 200

### Step 13: 요청 로깅 미들웨어

- 로그 형식: `POST /api/v1/auth/login 200 23.4ms`

**수정 파일**: `app/main.py`
- `app.add_middleware(RequestLoggingMiddleware)` 1줄 추가 (CORS 아래)
